# E2E Audit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce `docs/superpowers/audits/2026-04-08-e2e-audit-report.md` — a severity-tagged finding list covering every feature in `CLAUDE.md` across all three roles, written by code-auditing the existing implementation and runtime spot-checking the parts code review can't verify.

**Architecture:** Each feature in the inventory becomes one independent audit task. Each task writes its findings to a fragment file under `docs/superpowers/audits/fragments/NN-feature-slug.md` so tasks can run in parallel without merge conflicts. A final assembly task concatenates fragments, computes the summary header, and emits the report. No application code is modified at any point.

**Tech Stack:** Read-only against Next.js 16 / React 19 / Prisma 6 / Auth.js v5 source. Runtime checks via `npm run dev` against the seeded local Postgres DB.

**Spec:** `docs/superpowers/specs/2026-04-08-e2e-audit-design.md`

---

## Conventions for every audit task

Every feature task (Tasks 2–17) follows the same shape:

1. **Read the relevant CLAUDE.md section** — extract the "what should this do" line.
2. **Open every file in the task's `Files` list** — pages, server actions, API routes, components.
3. **Walk the checklist** — each task lists the specific things to verify (auth, async params, validation, Prisma correctness, dead links, etc.).
4. **Optional runtime check** — only the items the task marks as runtime-required.
5. **Write the fragment** — append findings to `docs/superpowers/audits/fragments/NN-slug.md` using the format from spec §5.
6. **Commit** — `docs(audit): <feature> findings`.

**Finding format (copy this into every fragment):**

```markdown
### [SEV] Short title
- **Feature:** <feature>
- **Location:** <file:line> or "runtime"
- **Roles affected:** STUDENT | TEACHER | ADMIN | ALL
- **Expected (per CLAUDE.md):** <spec>
- **Actual:** <observed>
- **Repro:** <trace or steps>
- **Notes:** <optional>
```

**Severity:** 🔴 BLOCKER · 🟡 BUG · 🔵 POLISH · ⚪ NOTE (definitions in spec §5).

**If a fragment has zero findings**, write:
```markdown
## N. <Feature Name>
**What it should do:** <one-line spec summary>

_No findings._

**Coverage:** <mini route × role table>
```

**Never modify application code.** If the urge strikes, log it as a finding and move on.
